# AARX fresh-machine setup

## Required versions

| Component | Version |
|---|---:|
| Ubuntu | 22.04 / 24.04 LTS |
| Python | 3.12.x |
| Node.js | 20 LTS or 22 LTS |
| npm | 10+ |
| Java | OpenJDK 17 |
| Android SDK | API 35 |
| PostgreSQL | 16 with PostGIS 3.x |
| PgBouncer | 1.21+ |
| Redis | 7.x |
| Expo | 53 (locked in package.json) |
| React Native | 0.79.6 |
| Gradle | 8.13 wrapper included |

## Install Ubuntu dependencies

Run:

    sudo apt update
    sudo apt install -y python3.12 python3.12-venv python3-pip postgresql-16 postgresql-16-postgis-3 pgbouncer redis-server openjdk-17-jdk adb build-essential libpq-dev gdal-bin libgdal-dev

Install Node 20/22 LTS and Android Studio separately. Android Studio needs SDK Platform 35, Build Tools and platform-tools.

## Fast project bootstrap

    cp django/.env.example django/.env
    cp AARXUI/.env.example AARXUI/.env
    # Edit both .env files. Never add secrets to Git.
    ./bootstrap-local.sh
    ./start.sh

bootstrap-local.sh installs Django, AI and Expo dependencies, runs Django checks and migrations. System packages and cloud accounts are intentionally manual.

## Required local files

| File | Why |
|---|---|
| django/.env | Backend secrets, database, Redis, S3, Razorpay, mail |
| AARXUI/.env | Android API address: EXPO_PUBLIC_BASE_URL |
| AARXUI/google-services.json | Firebase Android configuration |
| django/venv2/, ai_service/venv/, AARXUI/node_modules/ | Generated by bootstrap; never commit |

Do not commit .env, Firebase JSON, AWS credentials, Razorpay secret, database password or Android keystore.

## Database and Redis

Local default is aarxuser@127.0.0.1:6432/aarxdb. Create once:

    sudo -u postgres psql
    CREATE USER aarxuser WITH PASSWORD 'change-this-local-password';
    CREATE DATABASE aarxdb OWNER aarxuser;
    \c aarxdb
    CREATE EXTENSION postgis;
    \q
    sudo systemctl enable --now postgresql pgbouncer redis-server
    redis-cli ping

Configure PgBouncer to route 6432 to PostgreSQL 5432. If you do not use PgBouncer, set DB_PORT=5432 in django/.env.

## Firebase Android config

1. Firebase Console: create/select project and add Android app.
2. Package must match com.anonymous.AARXUI, or change AARXUI/app.json first.
3. Download google-services.json.
4. Save it exactly at AARXUI/google-services.json.
5. This file is ignored by Git.
6. Rebuild Android after Firebase/native package changes. Metro reload alone is not enough.

## Android device run

Set EXPO_PUBLIC_BASE_URL in AARXUI/.env to the computer LAN IP, for example http://192.168.1.100:8000. A Wi-Fi phone cannot use the computer localhost.

    cd AARXUI
    npx expo run:android
    npx expo start --dev-client
    adb reverse tcp:8000 tcp:8000

Razorpay is native: Expo Go cannot open Razorpay Checkout. Use the rebuilt AARX development app or APK.

## Credentials needed for complete features

- Razorpay: key ID, key secret, subscription/emergency webhook secrets.
- AWS S3: IAM access key, secret, bucket and region.
- Firebase: Android JSON and Firebase project setup.
- Email OTP: SMTP values.
- Maps: Google Places and/or Mapbox token.
- WhatsApp OTP: optional WhatsApp Graph values.

## start.sh services

- Django + Channels WebSocket: port 8000.
- AI service: port 8010.
- Redis broker and Channels: Redis DB 0.
- Django cache/rate limiting: Redis DB 1.
- Celery worker: notifications and default queues.
- Expo Metro: dev-client mode.

## Validation

    ./django/venv2/bin/python django/manage.py check
    ./django/venv2/bin/python django/manage.py migrate --plan
    redis-cli ping
    cd AARXUI && npm exec tsc -- --noEmit

## Production checklist

- Set DJANGO_DEBUG=false.
- Restrict DJANGO_ALLOWED_HOSTS and CORS origins.
- Enable HTTPS redirect, trusted proxy and HSTS.
- Use private/password-protected PostgreSQL and Redis.
- Restrict AWS IAM, Firebase API keys and Razorpay webhook secrets.
- Register both Razorpay webhook URLs.
- Run python manage.py check --deploy.
- Back up PostgreSQL, monitor failed Celery tasks and rotate credentials.
